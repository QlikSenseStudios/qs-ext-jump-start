# Project Setup Patterns

Reusable approaches for initializing any type of project.

## Universal Setup Steps
1. **Define Purpose**: Clear problem statement and goals
2. **Choose Structure**: Directory organization for project type
3. **Initialize Version Control**: Git repository with appropriate .gitignore
4. **Document Foundation**: README.md with purpose and usage
5. **Establish Quality**: Basic standards and validation approaches

## Context Loading for Setup
```
context.global.md → context.local.md → patterns/setup.md → domains/{type}.md
```

## Adaptive Elements
- Adjust structure based on project domain
- Select tools appropriate for project type
- Customize documentation for audience
- Configure quality tools for language/framework

## Common Artifacts
- README.md (purpose, usage, contribution)
- .gitignore (appropriate for project type)
- Basic directory structure
- License file (if sharing)
