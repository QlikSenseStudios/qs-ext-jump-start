# Reusable Patterns

Reusable approaches and procedures that can be applied across any project type or domain. Load only the patterns relevant to your current task.

## Available Patterns

- [**setup.md**](setup.md) — Project initialization and setup patterns

*Add new patterns as your project needs them. See [docs/Patterns.md](../docs/Patterns.md) for how to add a pattern.*

## Usage

Patterns provide:
- Step-by-step approaches for common challenges
- Decision guidelines applicable across domains
- Procedures that work regardless of project type

```
context.global.md → patterns/{name}.md
```

Combine with domain and project context for complete guidance:
```
domains/{type}.md + patterns/setup.md → domain-specific setup approach
```
