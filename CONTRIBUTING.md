# Contributing to QS-Ext-Jump-Start

This guide is for contributors to the template itself — bug fixes, improvements, and new features that benefit all users of the template.

## How to Contribute

1. Fork [QlikSenseStudios/qs-ext-jump-start](https://github.com/QlikSenseStudios/qs-ext-jump-start) and clone your fork
2. Install dependencies: `npm install`
3. Set up a Qlik environment for test validation: [Qlik Cloud](./docs/QLIK_CLOUD_SETUP.md) · [Enterprise](./docs/QLIK_ENTERPRISE_SETUP.md)
4. Make changes — follow the code style and organization in `src/` and `test/`
5. Run `npm run lint` and `npm test` — all checks must pass
6. Open a pull request to `QlikSenseStudios/qs-ext-jump-start` with a clear description of what changes and why

## Guidelines

- Keep pull requests focused — one concern per PR
- Update documentation if the change affects usage or functionality
- Write clear commit messages

## Reporting Issues

Open an issue in [GitHub Issues](https://github.com/QlikSenseStudios/qs-ext-jump-start/issues) with details and steps to reproduce.
